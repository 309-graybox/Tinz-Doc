# Описание
___
Прыжок является элементом вертикального передвижения. 
Внутри игры механика прыжка решает 4 базовых задачи:

1. **Перемещение по локациям:** С помощью прыжка игрок может передвигаться там, где он не мог бы пройти передвигаясь пешком, в такие места входят разнообразные обрывы и высокие препятствия.
   
2. **Избегание опасности:** Прыжком можно избежать атаки противника, или последствия активации ловушки.
   
3. **Понимание игроком ритма игры:**  Прыжок это одно из первых действий которое совершает игрок начиная играть, это механика тесно связанна с левел дизайном и дизайном противников.  Пейсинг и тайминг прыжка дает базовое понимание скорости  и ритма игрового процесса;
   
4. **Снятие напряжения:** Правильно подобранный ритм прыжка и анимация, помогают игроку бороться с игровым стрессом. 

Для обеспечения максимального контроля прыжка используются следующие механики:

- [[#Адаптивный прыжок]];
- [[#Буферизация ввода прыжка]];
- [[#Время Койота]];
- [[#Огибание коллизий]].
# Как работает обычный прыжок
___
При нажатии [[Jump (Input)|инпута прыжка]], вертикальной скорости персонажа (в обратном направлении от [[Гравитация#Активная сила притяжения|текущей гравитации]]) присваивается значение [[Прыжок (params)#2. JumpSpeed|начальной вертикальной скорости]]. При выполнении прыжка вертикальная скорость персонажа переопределяется, независимо от предыдущего значения.

Удержание инпута влияет на высоту прыжка через механизм [[#Адаптивный прыжок|адаптивного прыжка]].
Каждый кадр вертикальная скорость уменьшается под влиянием текущей гравитации.

Прыжок срабатывает только в случае если под персонажем есть опора - поверхность отмеченная соответствующей маской. 

Прыжки не оказывают прямого влияния на [[Бег и Ходьба|горизонтальное передвижение]].
# Адаптивный прыжок
___
Данная механика позволяет совершать прыжки не в полную силу. 

Если во время прыжка отпустить инпут прыжка, до того как вертикальная скорость уменьшится до [[Прыжок (params)#5. AdaptiveJumpTreshold|порогового значения]] - то вертикальная скорость персонажа мгновенно уменьшится на [[Прыжок (params)#4. AdaptiveJumpDamping|фиксированный процент]]. Из за этого персонаж продолжит некоторое время по инерции лететь вверх, но потеряет свою скорость намного быстрее, и соответственно быстрее приземлится.

# Буферизация ввода прыжка
___
Механика прыжка должна поддерживать [[Буферизация инпутов|буферизацию ввода]] прыжка, для обработки случая преждевременного нажатия на кнопку прыжка (до того как персонаж коснулся поверхности). 

Для буферизации инпута прыжка используется следующий [[Прыжок (params)#6. JumpInputBufferTimer|таймер буфера]].

# Время Койота
___
Механика времени койота решает сразу несколько задач:
- Решение проблемы сопоставления визуального представления и представления обработки пересечений
- Создание люфта для компенсации технической задержки ввода

Механика реализуется через добавление [[Прыжок (params)#7. CoyoteTime|таймера]] с обратным отсчетом, который запускается в момент потери опоры под персонажем. Пока этот таймер не истек инпут прыжка должен обрабатываться точно также, будто под персонажем есть опора.

# Огибание коллизий
___
Если во время прыжка персонаж сталкивается с препятствием, система может попытаться сместить его вперед на [[Прыжок (params)#8. ForwardPushLength|небольшое расстояние]] по направлению движения, чтобы сохранить вертикальное движение, если впереди есть свободное пространство.
